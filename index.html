Muon Protocol

Muon

/mjuːɒn/

( from the Greek letter mu (μ) used to represent it)

Muon is an elementary layer0 solution turning blockchain technologies into an ecosystem. The muon protocol, like an elementary particle, is one of the base-layer building blocks of our new decentralized world. It consists of a large number of nodes that sign valuable data upon request and push sensitive data to blockchains. We imagine the world computer like this: Ethereum is the immutable & static data storage (ROM), while muon mimics random access memory (RAM).

A Cyber-industrial High-Performance Oracles

Pythia is a layer 0 solution to turn blockchain technologies into an ecosystem. It consists of a large number of nodes that sign valuable data upon request and push sensitive data on to blockchains.

This is a live demo of the pythia stocks oracle. Type in TSLA to get a production ready pythia-signed live price from finnhub, ready to be used on-chain.*

Trust

Upon the web3 request and price claim, the Pythia network will internally verify the price/data that the oracle has signed and keeps the stake that the node locked if there is any disputes.

speed

Upon your request for a trade in a DApp, a HTTPs request is sent to the Pythia Node network and within a few milliseconds (current demo is running on 500 MS) you get a response with signatures that you can merge with your transaction into a message on any EVM chain.

compasibility

APP classes are defined, and every app can customize their class for the fully immersed Oracle solution on any chain. Pythia is therefore a fully customizable layer 0 and totally chain-agnostic.

Pythia SDK

A web3 JS library will be provided for developers to import in their DApps. The SDK will enable developers to initiate transactions through their DApps without referring users to the Pythia website. This enables the creation of large numbers of DApps using Pythia nodes. DEUS finance has already built an oracle for providing live stocks prices based on Pythia.

MUON NODES

Muon can have only one or several roles. Some nodes sign data and push it to blockchain. Some others observe the data on blockchain and place disputes wherever they see anomalies. Other nodes can only take the hosting role and use IPFS to save data.

MUON Token

The Muon token is the building block of the whole ecosystem and acts as the vehicle of service. It is the first implementation of the μRC20 token standard. A token that is on multiple chains with its balances and transactions being saved on multiple chains. Muon nodes combine all data from all chains before allowing a transaction to run or fail.

Muon token covers transaction/infrastructure cost to nodes, it is sent along with the request for the data in order for the node to do the actual verification process.

The transaction fee is mainly used to actually pay for the insured amount the nodes puts at stake to verify the transaction, and secondary for the actual node infrastructure.

The Muon protocol aims to be low in actual costs for infrastructure more like an decentralized AWS in comparsion to common blockchain infrastructures.
